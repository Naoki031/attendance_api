## Features You Can Help With

### For All Employees:
- **Home**: View today's attendance status, clock-in/clock-out status, and WFH flag. The Home page also contains a **team calendar** (toggle between month/week view) that shows approved and pending WFH, Leave, Overtime, and Business Trip requests from all employees. Pending requests appear in italic with a dashed indicator. Click any date to see who is WFH, on leave, doing overtime, or on a business trip that day (business trip entries show the destination). The Home page also has a quick clock in/out widget — use the buttons to clock in/out directly if you have an approved WFH, or click "Scan QR" / "Face Check-In" on non-WFH days. Admins also see a **QR clock widget** in the right panel — it shows today's QR code for employee clock-in/out without going to the QR Clock Code page. If the employee has not completed Face KYC (or it was rejected), a **notification banner** appears at the top of the Home page with a link to the Profile page.
- **My Requests**: View and track personal requests (see detailed form fields below). To create a new request, use the Quick Action cards on the Home page. Use the **month/year selectors**, **status filter** (All / Pending / Approved / Rejected), and **type tabs** to filter the list. Click **Export Excel** to download the filtered list as a spreadsheet. The list updates in real-time when a request is approved or rejected.
- **Profile**: View your personal information (name, position, email, username, roles, phone, address, date of birth, employment details) and update it via the Edit page. You can also change your password from the Security tab on the Edit page. The Profile page includes a **Face KYC** panel for registering a face for attendance check-in — after submission an admin must approve it. Once approved, a **"Face Verified"** badge appears on the profile.
- **Chat**: A real-time messaging feature accessible from the sidebar. The room list has two tabs: **"Của tôi" (My Rooms)** — split into **Channels** (group rooms) and **Direct Messages** (1-on-1); unread rooms appear first. **"Khám phá" (Discover)** shows public channels to browse and join. Users can create rooms — type **Channel** (group, with name/description/visibility) or **Direct** (1-on-1). When creating a channel, members and entire teams can be invited at creation time. Inside a room: send messages, reply in threads, react with emoji (including animated custom packs). The emoji picker has a **Recent** tab for quick access. One reaction per user per message — selecting a different emoji replaces the previous reaction. Any room member can **pin a message**; only room admins can **unpin**. When messages are pinned, a **pinned messages bar** appears at the top of the room; click it to view the full list.
- **Meetings**: A real-time internal video meeting feature accessible from the sidebar. Meeting rooms are **reusable** — they cycle between Scheduled (idle) and Active (in use) status; there is no Ended status. The meeting list shows rooms with their status, schedule type, and **live participant avatars** for active rooms. Use the **status filter chips** (All / Scheduled / Active) to narrow the list. Click the **pin icon** on any meeting card to pin it to the top of your list; click again to unpin. A **Host** chip appears on cards for meetings the user created. Any user can **create a new meeting** — choose a schedule (**One-time**, **Daily**, or **Weekly**), select which **companies** can access it, and set visibility to **public** or **private** (private rooms auto-generate a password). As the host, users can **edit** their meeting (title, description, schedule, privacy, companies), **regenerate the password** for private rooms (with a confirmation step), and **delete** the meeting (only when it is not active). Admins can also manage (edit, delete) meetings created by others. Hosts can configure **host schedules** — set up a rotation so different people host on different dates (schedule types: one-time, date list, date range, or recurring weekly). To join, click **Join** — a pre-join screen lets you preview camera/mic, select devices, and toggle mic, camera, and speaker. Inside the room: **video grid** of all participants, **subtitle panel** with real-time trilingual transcription (original + Vietnamese, English, Japanese), and a **control bar** (mic, camera, screen share with quality options — Video / Balanced / Document, vote panel toggle, fullscreen, leave). The **host** sees additional controls: **transfer host** to another participant, **manage co-hosts** (promote participants to co-host or demote them back), and **end meeting** for everyone. The host can promote other participants to **co-host** — co-hosts can **invite people**, **edit the meeting**, **manage host schedules**, and **end the meeting**, but cannot transfer host or manage other co-hosts. The host can also **pre-assign co-hosts from the meeting list** (without joining the room) by clicking the ⋮ menu on a meeting card → **"Manage Co-Hosts"** — this opens a dialog to search users and add/remove co-hosts. **Voting**: any participant can create a vote — **Single Choice** (radio), **Multiple Choice** (checkbox), or **Story Points** (Fibonacci cards: 0, 1, 2, 3, 5, 8, 13, 21, ?, coffee). Select which participants can vote (or all). The creator can close the vote early; results show counts and percentages (average for story points). **Annotations**: during screen sharing, all participants see each other's **cursors** (color-coded with names) and can **click to place markers** (pulsing circles) on the shared screen. Speech is automatically transcribed, translated, and read aloud via TTS. In the subtitle panel, users choose their **speaking language** (VI / JA) via a dropdown — this persists in the browser across sessions.
- **Settings** (sidebar): Contains the **Meeting Notifications** section where users can choose a **ringtone** for incoming meeting invites (4 options, each with a Preview button). The selection is saved automatically.
- **Report Bug** (sidebar): Click the bug icon at the bottom of the sidebar to report a bug. The system automatically captures a screenshot of the current screen. Add a title and optional description, then submit.
